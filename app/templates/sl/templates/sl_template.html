Roster Comparison

This tool allows you to compare the data in the eCase Roster with the data in RosterOn

For the eCase data, run the report "Mentoring Roster Report", through the report generator.

For the RosterOn data run the report “Allocations by Employee”, through RosterOn as a "CSV with Headers". Once the report has been generated make sure you open it in Excel and then re-save it.

As the naming conventions in RosterOn and eCase are different, the tool requires a table to translate between the two. Sometimes you may find that the tools tells you that some staff or clients are missing.

Progress Note Check

This tool allows you to check who has entered their Progress Notes for shifts worked.

For the Roster Data, run the report "Mentoring Roster Report", through the report generator.

For the Progress Note data, run the report “Mentoring Progress Notes” through the eCase Report generator. Ensure that you retrieve all progress notes entered from the first date included in your Roster Report.

The report generates a list of “Missing Progress Notes” (where there is no progress note entered by that staff, on that service date, for that client), and “Time Errors” (where the staff has entered a progress note on that day, for that client, but the timings do not match).

Use the email button to generate an email to send to the staff.

PV Logsheet

This tool will allow you to generate a report to review staff's entered mileage as per their progress notes.

For the Progress Note data, run the report “Mentoring Progress Notes” through the eCase Report generator.

The report generated includes all progress notes within the dates included, that have mileage entered, and where the staff has indicated non-fleet vehicle usage.

Budget Overview

This tool will show you the agreed, used, scheduled and unallocated funding for a client. This report excludes ABT and Transport related funding when calculating the budget.

The Billing File is generated by running the “AAA_NDISByClient” report from the eCase report generator, for the specific client.

The Budget File is generated by running the “Quoted Budget” report from the eCase report generator. Ensure that you have the currently active agreement, and enter it into the “Version” field for the report filters.

The Schedule File is generated by running the “Scheduled Services” report from the eCase report generator, for the specific client.

Overview

The overview gives you a brief description of the current status of the specific client’s funding.

Quote Calculator

This tool allows you to add services, using specific NDIS codes, to see how that will impact on the clients budget.

Enter the number of hours the client wants the service to be in “Hours”.

Enter the number of reoccurring shifts the client would like under “Occurrences”.

Billing Report

This tool allows you to generate a report that compares the eCase Roster against entered Progress Notes, within information on what shifts are ready to be billed, and which require manual attention/adjustments.

Running the Report

For the Roster Data, run the report "Mentoring Roster Report", through the report generator.

For the Progress Note data, run the report “Mentoring Progress Notes” through the eCase Report Generator. Ensure that you retrieve all progress notes entered from the first date included in your Roster Report.

Interpreting the Report

Below is a list of the different headings within the report, and and explanation of the data.

Client The full name of the client receiving the service
Description The eCase billing code
Program Type The type of support provided
Service Date The date the service occurred as per the eCase Roster
Start Date Time The start date and start time as per the eCase Roster
End Date Time The end date and end time as per the eCase Roster
Instructions Instructions entered into the eCase Roster by Mentoring Support Leaders. NOTE: this is generally not relevant to billing, however can be useful when addressing errors as it should give an overview of what the shift is supposed to be.
Allocated Carer The name of the Support Worker as per allocations in the eCase Roster
StartTime The start time of shift as per Support Worker progress note
EndTime The end time of shift as per Support Worker progress note. NOTE: This is the end of their rostered shift, NOT face-to-face time or necessarily billable time
Roster Duration The time in minutes for the Roster Duration (i.e., the time the staff is rostered and being paid for)
Service Duration The time in minutes for the Charge Duration (i.e., the time that is going to be billed)
Notes The progress note entered by the Support Worker
Vehicle The type of vehicle being used (F = Fleet, H = House, P = Personal / Private, N/0/Blank = No vehicle. NOTE: As this is a free text-field, other letters may appear if the Support Worker enteres something else. If that is the case, please refer to the "Notes" column to verify what the Support Worker entered.
Kms Travelled Kms of ABT as entered by Support Worker in Progress Note. NOTE: This does not currently take into account whether the client has agreed to ABT, nor whether the reported Kms are within the agreed amount.
chargeDurationStatus Indicators of whether the difference between the Roster Duration and Charge Duration meets expectations based on Type of Vehicle used.
progNoteStatus
  • Accepted
    • Progress Note Date, StartTime, EndTime and Staff match the roster
  • TimeError
    • Progress Note Date and Staff match roster, but Timings are incorrect
  • Code Error
    • The billing code does not match the day of week / time of service
  • Missing
    • The allocated staff member has not entered a progress note with a service date that match the roster
Billing Status Indicates whether the shift is Accepted for billing based on the chargeDurationStatus and progNoteStatus.
